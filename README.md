# BritishAirways_Simulation
Airline Booking Customer Behavior AnalysisThis repository contains a Python Jupyter Notebook dedicated to analyzing and preprocessing customer booking data for an airline. The goal of this project is to clean, explore, and prepare a dataset of 50,000 booking records to understand customer behavior and predict successful bookings (booking_complete).
📌 Project OverviewThe dataset contains information about customer flight bookings, including booking channels, flight trip types, length of stay, purchase lead times, and optional services requested (extra baggage, preferred seats, in-flight meals).This project performs crucial Data Cleaning, Feature Encoding, and Exploratory Data Analysis (EDA) to prepare the data for predictive machine learning models.📊 Dataset FeaturesThe dataset (customer_booking.csv) contains 14 key features:Feature NameDescriptionnum_passengersNumber of passengers in the bookingsales_channelChannel used to book (Internet, Mobile)trip_typeType of trip (RoundTrip, OneWay, CircleTrip)purchase_leadNumber of days between booking date and travel datelength_of_stayDuration of stay at the destination (in days)flight_hourHour of the flight departureflight_dayDay of the week of the flight departurerouteFlight route (e.g., AKLDEL)booking_originCountry where the booking was madewants_extra_baggageIndicator (1/0) if the customer wants extra baggagewants_preferred_seatIndicator (1/0) if the customer wants a preferred seatwants_in_flight_mealsIndicator (1/0) if the customer wants in-flight mealsflight_durationDuration of the flight (in hours)booking_completeTarget Variable (1 = Booking completed, 0 = Booking incomplete)🛠️ Data Preprocessing & WorkflowKey tasks executed in the Jupyter Notebook:Google Colab File Upload: Dynamically uploads and loads the airline dataset.Data Inspection: Checking data types (.info()), statistical summary (.describe()), and null values (.isnull().sum()).Handling Duplicates: Identified and dropped 719 duplicate records, reducing the dataset size to 49,281 unique records.Categorical Encoding:Mapped flight_day from string representations (Mon, Tue...) to numerical labels (1, 2...).Used Category Codes (cat.codes) to label-encode other object/categorical columns such as sales_channel, trip_type, route, and booking_origin.EDA (Exploratory Data Analysis): Initiated data visualization and distribution analysis on the clean dataset.⚙️ Requirements & InstallationTo run this notebook locally, install the required libraries:Bashpip install pandas matplotlib seaborn
